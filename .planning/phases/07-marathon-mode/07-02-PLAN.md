**3. After each search completes in Map.tsx draw handlers (mouseup / touchend), promote the active draw rectangle to a zone rectangle:**

The draw completes in two places: `onTouchEnd` and the `addListenerOnce("mouseup")` handler. In both places, after `searchBusinessesInArea(bounds)` is called:

```typescript
// After searchBusinessesInArea(bounds) — promote rect to zone pool if marathon mode
const store = useStore.getState();
if (store.marathonMode && areaRectRef.current) {
  // Assign a temporary id; the zone id from discover-search will be set after async search
  // Use a stable approach: store the rect temporarily, then re-register after zone is known.
  // Simpler: just keep areaRectRef.current mapped by index/timestamp as a display artifact.
  // The rectangle is already on the map — set a distinct "zone" style and move to pool.
  areaRectRef.current.setOptions({
    strokeColor: "#D4712A",
    strokeWeight: 1.5,
    fillColor: "#D4712A",
    fillOpacity: 0.05,
    zIndex: 1,
  });
  const zoneKey = Date.now().toString();
  marathonZoneRectsRef.current.set(zoneKey, areaRectRef.current);
  areaRectRef.current = null;  // detach from areaRectRef so next draw starts fresh
  // Re-enter discover draw mode for the next draw in marathon session
  enterDiscoverMode();
} else {
  stopDrawing();
}
```

**IMPORTANT:** The re-enter call `enterDiscoverMode()` after marathon draw REPLACES the existing explicit `stopDrawing()` call at the end of desktop mouseup and mobile touchend handlers when marathonMode is true. When marathonMode is false the existing `stopDrawing()` path remains unchanged.

Adjust the mouseup handler — the existing code calls `stopDrawing()` unconditionally before the search. In marathon mode, `stopDrawing` should still be called to remove the draw listeners (we're promoting the rect, not continuing to drag). The re-`enterDiscoverMode()` call after the promotion re-registers fresh draw listeners for the next zone.
